Important information

Refunds

Even if Unity says "We've got you covered", i still have the final word (this is the same for any asset on the store). I will authorize a refund for you only if the asset advertises something that it is not, for example, if some feature is not there, or doesn't even work (mistakes happen). If that's the case, then you can send me an email and ask for a refund, no problem at all.

That being said, assets on the Unity asset store (at least mine) do not follow a "try before buy" policy. Once you clicked the buy button there is no going back, so please make sure you are buying the asset you want. That's why i'm open for questions 24/7, i can give you code examples, show you videos/gifs, whatever you need to make sure you're purchasing the asset you need for your project.

Support

Once you buy any asset you will received free support. Please when asking for something try to be very clear and organized, indicate your version of the asset and your Unity editor.

Regarding support, questions about the last version of an asset will have a higher priority. So, if your asset version is old, and your asking me about an issue that has been fixed already, then i would encourage you to update. If that's not an option i can still help you of course (sometimes developers need to stick with a particular version, and that's fine).

Readme files

A common problem is that users tend to ignore any readme file included with the asset, especially when there is demo content involved. Please, before asking for support, or going in rant mode to leave a bad review on the asset store, take a breath and read the readme file. I tend to put a "OPEN ME" folder with a "README" text file. Don't worry, it won't make you lose your whole day, only half a minute.

Editor support

Never purchase some asset assuming it will work with an unofficial version of Unity. If you still want to use some of my assets with an older version of Unity, ask first! Otherwise i won't give you any support.

What if Unity 20XY.Z comes out tomorrow, you import the asset and something doesn't work, or a console error appears, or something unusual happens. Well, first don't freak out, that could happen, especially since Unity is "evolving" year after year. If this happens to you the best thing you can do is to send me an email describing your situation.
